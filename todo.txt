Observations:

- when document is very small, there are few topics found, so large
(50%) summaries do not have any opportunity to get glue
	solution: use fallback approach (initial paragraphs), or
		include last sentence as a topic boundary

- 25-30% factors appears to work best (i.e., generate the correct size
summary) for documents under 70 sentences, regardless of the length of
the summary

- for > 70 sentences, 10-20% factors appears to generate the correct
length summary

- factors above 30% do not appear to be useful in any case

General comprehensiveness/coherence:

< 70:
musicians is no good for length < 30%
		25% is best for length 30% and 50%

getallglue.ksh can be made vastly more efficient with the following
reimplementation: 

0) input: length l, list of representative sentences, alpha beta gamma
1) for each pair of representative sentences i, j
	getallglue(i, j, 0)
2) num = `wc -l < $corpus.reps`
3) head -l $corpus.reps > $corpus.sumindex
4) l -= num
5) z = 1
6) while l > 0
	num = `wc -l $corpus.$z`
	if num > l
		sort +1nr $corpus.$z | head -$l | cut -f2
		exit
	else
		cut -f2 $corpus.$z
		z++
	fi


function getallglue(x, y, z)
	g = bestglue(i,j)
	s = sim(i,j)
	if g exists
		echo $s $g >> $corpus.summary.$z
		call getallglue(i, g, z+1)

"DUC-2001:   Procedure for Human Comparison of Summaries"
(assessment_instructions.doc) requested participants to evaluate
cohesion without respect to coherence:

\begin{quote}Cohesion---do the sentences fit in as they should with
the surrounding sentences? Don't worry about the overall structure of
the ideas. Concentrate on whether each sentence naturally follows the
preceding one and leads into the next.\end{quote}
